¿Qué es la regresión en ML?

La regresión predice un valor numérico continuo a partir de variables de entrada. El objetivo es encontrar la función f(X) que minimiza el error entre predicciones y valores reales.

ŷ = f(X) + ε donde ε ~ N(0, σ²)

Regresión lineal (OLS)

Relación lineal entre features y target. Base de todo. Solución analítica exacta.

OLSRápido

Regularización

Ridge (L2), Lasso (L1), ElasticNet: penalizan coeficientes para evitar overfitting.

L1 / L2Anti-overfitting

No lineal

SVR, Random Forest, Gradient Boosting: capturan relaciones no lineales.

FlexibleAlta precisión
💡 Regla de oro: siempre empieza con Regresión Lineal como baseline antes de probar modelos más complejos.

Tipos de regresión

1. Regresión Lineal (OLS)

ŷ = β₀ + β₁x₁ + … + βₙxₙ → min Σ(yᵢ - ŷᵢ)²
✓ Interpretable✓ Rápido✗ Solo relaciones lineales

2. Ridge (L2)

min Σ(yᵢ - ŷᵢ)² + α·Σβⱼ²
✓ Multicolinealidad✓ Estabilidad~ No selecciona features

3. Lasso (L1)

min Σ(yᵢ - ŷᵢ)² + α·Σ|βⱼ|
✓ Selección automática✓ Alta dimensión✗ Grupos correlados

4. ElasticNet (L1 + L2)

min Σ(yᵢ - ŷᵢ)² + α·[ρ·Σ|βⱼ| + (1-ρ)·Σβⱼ²]
✓ Lo mejor de ambos~ Más hiperparámetros

from sklearn.preprocessing import StandardScaler from sklearn.pipeline import Pipeline pipeline = Pipeline([ ('imputer', SimpleImputer(strategy='median')), ('scaler', StandardScaler()), ('model', Ridge(alpha=1.0)) ])

Optimización con GridSearchCV

from sklearn.model_selection import GridSearchCV param_grid = { 'model__alpha': [0.001, 0.01, 0.1, 1.0, 10.0, 100.0] } gs = GridSearchCV(pipeline, param_grid, cv=5, scoring='r2', n_jobs=-1) gs.fit(X_train, y_train) print(f"Mejor alpha: {gs.best_params_}") print(f"Mejor CV R²: {gs.best_score_:.4f}")

Validación Cruzada (K-Fold)

Divide los datos en K grupos. Entrena K veces usando K−1 folds y valida en 1. Score final = promedio de los K scores.

Estrategias de CV

MétodoCuándo
KFold (K=5/10)Regresión estándar
TimeSeriesSplitDatos temporales
RepeatedKFoldMás robustez estadística
Leave-One-OutDatasets muy pequeños

Código

from sklearn.model_selection import ( cross_val_score, KFold) kf = KFold(n_splits=5, shuffle=True, random_state=42) scores = cross_val_score( model, X_train, y_train, cv=kf, scoring='r2') print(f"R² μ={scores.mean():.3f}" f" σ={scores.std():.3f}")
⚠️ Regla anti data-leakage: nunca uses el test set para seleccionar hiperparámetros. Toda optimización va dentro del CV en el train set.

¿Cuándo usar cada modelo?

ModeloVentajasLimitacionesContexto ideal
Lineal (OLS)Interpretable, rápidoSolo relaciones linealesBaseline siempre
RidgeEstable con multicolinealidadNo hace selecciónMuchas variables correladas
LassoSelección automáticaInestable grupos correladosAlta dimensión, sparsity
ElasticNetCombina Ridge+LassoMás hiperparámetrosAlta dim + grupos correlados
SVRNo lineal, robustoLento N grandeDatasets medianos, no lineal
Random ForestRobusto, paraleloMenos interpretableProducción, datos mixtos
Gradient BoostingState-of-the-artRiesgo overfittingCompetencias, grandes datasets
1

Empieza siempre con Regresión Lineal como baseline

2

Si multicolinealidad alta → Ridge. Si quieres selección → Lasso/ElasticNet

3

Si la relación no es lineal → SVR o Random Forest

4

Si necesitas máxima precisión → Gradient Boosting (XGBoost, LightGBM)

5

Valida siempre con CV y selecciona hiperparámetros antes de reportar en test
